state : 컴포넌트의 상태state를 저장하고 변경할 수 있는 동적 데이터. 변경될 가능성이 있는 데이터를 다룰 때 사용. 
클래스형 컴포넌트에서 사용하는 개념. 객체형 데이터

this.state는 현재 클래스에서 선언된 state.

set.state()함수를 사용해서 state값을 변경해야 함.

리액트는 state가 변경되면 render()함수를 다시 실행하여 변경된 state를 화면에 출력함. 그런데 state를 직접 변경하는 경우에는 render()함수를 다시 실행하지 않는다. 그래서 리액트는 state를 직접 변경할 수 없도록 함.

setState()함수가 동작하면 -> state가 새로운 값으로 바뀌고 -> 이어서 render()함수를 동작시켜 -> 화면을 업데이트 시킴

setState()함수의 인자로 함수를 전달하면 성능 문제 없이 업데이트 할 수 있음.
리액트는 이전 state와 새로운 state를 비교하여 바뀐 데이터만 업데이트 함.

리액트에서 마운트(Mount)로 분류하는 생명주기 함수. 실행순서.
componentDidMount()함수는 컴포넌트가 처음 화면에 그려지면 실행되는 함수.
constructor() -> render() -> componentDidMount()

리액트에서 업데이트(Update)로 분류하는 생명주기 함수. 실행순서
setState() -> render() -> componentDidUpdate()
